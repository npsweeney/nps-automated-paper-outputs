# Shovel-Ready Trap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a full `shovel-ready-trap` regional-science paper project with OpenAlex corpus tooling, manuscript, references, evidence ledger, reviewer automation, output bundle, website page, SSRN package, and Overleaf status.

**Architecture:** Keep the reusable validation/reviewer machinery in `pipeline/` and add paper-specific OpenAlex/literature/data scripts under `papers/shovel-ready-trap/scripts/`. Register the paper in `meta/papers.yml` so the existing meta runner can publish to `papers-out/` and regenerate the GitHub Pages site. Use cached fixtures and normalized JSONL for deterministic tests, while live OpenAlex fetching remains opt-in and reproducible.

**Tech Stack:** Python 3 standard library, JSON-valid YAML, LaTeX, Markdown, BibTeX, POSIX shell, `unittest`, OpenAlex HTTP API.
